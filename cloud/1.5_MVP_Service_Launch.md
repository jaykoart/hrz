# Phase 1.5: MVP 서비스 구축 및 출시 준비

**문서 최종 수정일:** 2025년 11월 29일

## 1. 개요

지금까지 구축한 하드웨어 및 소프트웨어 인프라 위에 고객에게 판매할 수 있는 최소 기능 제품(MVP, Minimum Viable Product)을 구축하고, 서비스 출시를 위한 최종 준비를 마치는 단계입니다. 초기 서비스는 '웹 호스팅'과 '가상 서버(VPS) 호스팅'에 집중합니다.

## 2. 웹 호스팅 서비스 구축 (cPanel/WHM 활용)

산업 표준인 cPanel/WHM을 사용하여 안정적이고 관리가 용이한 공유 웹 호스팅 서비스를 구축합니다.

### 2.1. cPanel용 가상머신(VM) 생성
1.  **Proxmox VE 웹 인터페이스**에서 `node1` 선택 -> `VM 생성`.
2.  **일반:** 이름 `cpanel-server` 입력.
3.  **OS:** `스토리지`에서 `local` 선택, `ISO 이미지`는 `CentOS 7.9` 선택.
4.  **시스템:** 기본값 유지.
5.  **디스크:** `스토리지`는 `OS_SSD_RAID1`, `디스크 크기`는 `100`GB 입력.
6.  **CPU:** `코어` 수 `2` 입력.
7.  **메모리:** `4096`MB (4GB) 입력.
8.  **네트워크:** `브릿지`는 `vmbr0`, `모델`은 `VirtIO` 확인.
9.  VM 생성 후, 콘솔을 열어 CentOS 7을 기본 설치합니다. (네트워크 활성화 필수)

### 2.2. cPanel/WHM 설치
1.  **라이선스 구매:** cPanel 공식 스토어([https://store.cpanel.net/](https://store.cpanel.net/))에서 `cPanel Solo` 라이선스를 VM의 공인 IP 주소로 구매합니다.
2.  **SSH 접속:** PC의 터미널을 통해 `ssh root@<VM의 IP>` 명령으로 VM에 접속합니다.
3.  **설치 스크립트 실행:** 아래 명령어를 순서대로 실행합니다.
    ```bash
    yum update -y
    yum install -y perl
    cd /home && curl -o latest -L https://securedownloads.cpanel.net/latest && sh latest
    ```
4.  설치는 약 30분~1시간 소요됩니다. 완료 후 `https://<VM의 IP>:2087`로 접속하여 WHM 초기 설정을 진행합니다. (이메일 주소, 네임서버 등)

### 2.3. 호스팅 상품 패키지 생성 (WHM)
1.  WHM 인터페이스에서 `패키지` -> `패키지 추가` 메뉴로 이동합니다.
2.  **상품 1 (베이직):**
    - `패키지 이름`: `Basic_10G`
    - `디스크 공간 할당량 (MB)`: `10240` (10GB)
    - `월 대역폭 제한 (MB)`: `102400` (100GB)
    - `최대 이메일 계정`: `10`
3.  **상품 2 (비즈니스):**
    - `패키지 이름`: `Business_50G`
    - `디스크 공간 할당량 (MB)`: `51200` (50GB)
    - `월 대역폭 제한 (MB)`: `512000` (500GB)
    - `최대 이메일 계정`: `무제한`
4.  `추가` 버튼을 눌러 각 패키지를 저장합니다.

## 3. 가상 서버(VPS) 호스팅 서비스 구축

Proxmox VE의 템플릿 기능을 활용하여 고객 주문 시 VPS를 신속하게 생성하고 제공합니다.

### 3.1. VM 템플릿 생성
1.  **기본 VM 생성 및 설정:**
    - Proxmox VE에 `Ubuntu 22.04 Server`와 `Rocky Linux 9`의 VM을 각각 최소 사양으로 설치합니다.
    - 각 VM에 접속하여 `apt update && apt upgrade -y` 또는 `dnf update -y`를 실행하여 시스템을 최신 상태로 유지합니다.
    - SSH 접속 및 원격 관리를 위한 `qemu-guest-agent`와 프로비저닝 자동화를 위한 `cloud-init`을 설치합니다.
    ```bash
    # Ubuntu
    apt install -y qemu-guest-agent cloud-init
    # Rocky Linux
    dnf install -y qemu-guest-agent cloud-init
    ```
2.  **템플릿으로 변환:**
    - 기본 설정이 완료된 VM을 종료합니다.
    - 해당 VM의 마우스 오른쪽 버튼 메뉴에서 `템플릿으로 변환`을 선택합니다.

### 3.2. VPS 상품 구성
- **VPS-Standard-1:** 1 vCPU, 2GB RAM, 50GB NVMe SSD, 월 15,000원
- **VPS-Standard-2:** 2 vCPU, 4GB RAM, 100GB NVMe SSD, 월 30,000원
- **VPS-Premium-1:** 4 vCPU, 8GB RAM, 200GB NVMe SSD, 월 60,000원

### 3.3. 고객 주문 및 프로비저닝 (초기 수동 대응)
- 초기에는 고객이 이메일이나 문의 게시판으로 주문하면, 관리자가 직접 Proxmox VE에서 템플릿을 복제하여 VPS를 생성하고 정보를 전달합니다.
- **수동 생성 절차:**
    1.  템플릿 VM의 마우스 오른쪽 버튼 메뉴에서 `복제` 선택.
    2.  `모드`는 `전체 복제` 선택, 이름은 고객 식별 정보로 지정.
    3.  생성된 VM의 `하드웨어` 탭에서 상품 사양에 맞게 CPU 코어, 메모리, 디스크 크기 조정.
    4.  `cloud-init` 탭에서 사용자 이름, 암호 또는 SSH 공개키 설정.
    5.  VM을 시작하고 고객에게 IP 주소와 접속 정보를 전달.

## 4. 백업 서비스 상품화

### 4.1. 백업 정책 및 가격
- **웹 호스팅:**
    - **기본 제공:** 매일 새벽 2시, cPanel 계정 전체를 Proxmox Backup Server(PBS)로 자동 백업. 데이터는 최종적으로 AWS S3에 저장.
    - **보관 주기:** 매일 백업 7일, 주간 백업 4주, 월간 백업 3개월.
- **VPS 호스팅 (유료 부가 서비스):**
    - **주간 백업:** 주 1회 전체 VM 스냅샷 백업. (월 5,000원/VPS)
    - **매일 백업:** 매일 새벽 시간대 스냅샷 백업. (월 15,000원/VPS)

### 4.2. 백업 설정
1.  Proxmox VE의 `데이터센터` -> `백업` 메뉴로 이동.
2.  `추가` 버튼을 눌러 백업 작업을 생성합니다.
3.  `노드`는 `node1`, `스토리지`는 `pbs-main` (Phase 1.4에서 설정) 선택.
4.  `일정`은 `매일 02:00` 과 같이 설정.
5.  `선택 모드`를 통해 백업할 VM(cPanel 서버 등)을 지정.
6.  `활성화됨`을 체크하고 작업을 생성.

## 5. 서비스 출시 준비

### 5.1. 서비스 소개 웹사이트 제작
- **플랫폼:** WordPress + Elementor 페이지 빌더 사용.
- **필수 콘텐츠:**
    - 회사 소개
    - 웹 호스팅 상품 소개 및 가격, 신청 링크
    - VPS 호스팅 상품 소개 및 가격, 신청 링크
    - 기술 지원 문의 방법 (이메일, 게시판)
    - 이용 약관 및 개인정보처리방침
- **호스팅:** 제작된 웹사이트는 우리가 구축한 cPanel 서버의 첫 번째 고객사로서 직접 운영.

### 5.2. 초기 고객 유치 프로모션
- **대상:** 익산 지역 소상공인 온라인 커뮤니티, 전북대학교/원광대학교 창업 동아리, 디자인/개발 관련 학과 학생.
- **혜택:** '베이직' 웹 호스팅 또는 'VPS-Standard-1' 상품 3개월 무료 체험.
- **조건:** 서비스 이용 후기 작성 또는 SNS 공유.
